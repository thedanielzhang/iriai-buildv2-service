Proposed Execution Control Plane

This is not just a cleaner version of the monolith. It splits authority into durable modules: typed journal state, workspace and contract preflight, sandboxed candidate production, evidence-backed verification, typed failure routing, durable merge/checkpoint ownership, and guarded post-DAG completion.

Atomic Landing Constraint

No per-slice production rollout, no partial product-authoritative mode, and no automatic migration of active legacy features. The complete bundle is enabled only after the landing gate passes. Rollback stops admission or rolls back the complete feature while preserving typed audit rows and compatibility projections.

End-To-End Control And Data Ownership

The center rail is the normal candidate path. The left rail owns orchestration and persistent state. The right rail owns failure routing, compatibility readers, regroup overlays, and post-DAG/post-test guards.

then

Compiles required, allowed, and forbidden paths; acceptance criteria; verification gates; write-set ownership; and merge admission rules.

then
GATE REQUEST

Deterministic Preflight Gates

Workspace, contract, dependency, stale-context, path-scope, and projection checks block dispatch before model work when evidence is unsafe.

then
SANDBOX

SandboxRunner

Allocates isolated roots and captures immutable patch evidence. Product-content mutation of canonical repos waits for the merge queue; metadata-only workspace normalization is recorded separately.

then
RUNTIME

RuntimeDispatcher

Invokes Claude, Codex, or pool runtimes through a narrow journal facade. It records attempts and runtime failures, but cannot enqueue merge, route repair, commit, checkpoint, or treat dispatch success as integration success.

produces

Visible to legacy consumers as attempt evidence. It is not group completion, integration proof, or checkpoint proof.

then
APPROVAL GRAPH

  • Raw verifier, expanded lenses, aggregate verdict, stale-context rejection, and read-budget proof.
  • Pre-queue approval, post-apply gates, merge_gate, and separate checkpoint_gate.
  • Raw dag-verify:* projections and summary-only evidence are never authoritative alone.
Approved Candidate

DurableMergeQueue + GitService

  • Lease, claim, apply, rebase/conflict handling, and recovery.
  • Records merge_queue_task_coverage and merge_queue_repo_targets.
  • Runs post-apply gates before commit.
  • Writes merge proof, commit proof, no-dirty snapshot, checkpoint gate, checkpoint body evidence, and idempotent dag-group:* projection.
Typed Failure
ROUTE

FailureRouter Selects; Bounded Executors Act

FailureRouter classifies and selects the route. RepairService, Dispatcher, MergeQueue, WorkspaceAuthority, and GitService execute only the selected bounded action: product repair, contract repair, canonicalization repair, workspace repair, commit hygiene, verifier retry, runtime retry, merge retry, sandbox capture retry, sandbox cleanup, explicit operator-required, or quiesce. Workflow failures do not default to broad product repair.

Retry loops return to the earliest safe owner: workspace, contracts, sandbox, dispatcher, verifier, merge queue, or post-DAG gate.

Outer Guards And Consumers

SCHEDULING

RegroupOverlayService

Root DAG remains immutable. The typed active overlay and effective-DAG resolver define dispatch, resume, checkpoint coverage, rollback-before-first-wave, and post-test readiness. dag-regroup:* projections mirror that state for legacy readers. Scheduler feedback is advisory; safety outranks speed.

after all effective groups checkpoint
FEATURE GATES

PostDagGateService

Preserves code review, security audit, test authoring, QA, integration testing, final verifier, source-repo push, implementation report, backlog report, and completion notification. Group checkpoint is not final implementation approval.

then
GUARD

PostTestReadinessGuard

Allows observation only after effective-DAG completion, post-DAG gate approval, no active queue/repair work, and compatible projections. It never relies on root-DAG length alone.

side visibility
ADVISORY

Supervisor And Dashboard

No executor/control-plane/product mutation authority. Allowed writes are bounded audit, dedupe, display outbox, and explicitly configured guarded bridge actions. Typed snapshots outrank legacy fallback.

Proof Boundary Rules

SurfaceMeaning In Target Architecture
dag-task:* Attempt evidence projected from typed execution attempts. It never proves integration or checkpoint completion.
dag-verify:* Projection of verification graph nodes. Raw approval alone cannot admit merge or checkpoint.
dag-commit-failure:* Projection of typed commit-hygiene failure with bounded command output and a narrow route.
dag-group:* Written only after queue success, approved gate/checkpoint evidence, merge proof, commit proof, no-dirty proof, checkpoint body evidence, and idempotent projection/link success.
dag-gate:* Feature-level post-DAG gate projection backed by typed gate evidence. It is required before post-test observation.
dag-regroup:* Validated effective-DAG overlay projection. Root DAG remains immutable; rollback is guarded and forward-only after first work starts.
